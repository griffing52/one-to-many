# A cheap world model for trajectory perturbation

Goal: cheaply synthesize **variations** of a single recorded trajectory — base
height/lateral shifts, EE perturbations — and render the **ZED (third-person)**
and **RealSense (wrist)** views, to (1) augment data without re-collecting and
(2) simulate/recover from failures (the [a2l-pr](../../a2l-pr) idea).

## Division of labor between the two views

- **Third-person (ZED)** — *what / where*: scene context and the gross spatial
  relationship between gripper and object. Bag fidelity here can be coarse.
- **Wrist (RealSense)** — *how*: the fine gripper↔object interaction. This is
  where object fidelity matters (e.g. moving the bag to the box).

## The unifying trick: everything in the robot-base frame

Both cameras can be expressed in the **base frame**, so a single (perturbed)
joint/EE trajectory drives every layer:

- **ZED → base**: solved by PnP between the tracked green gripper (2D) and FK
  (3D) — no calibration target, ~7px reprojection. The ZED is fixed.
- **Wrist → base**: the wrist camera is rigidly mounted on the EE, so its pose is
  **FK of the camera link** at any (perturbed) joint config.
- **Robot**: URDF + FK at the (perturbed) joints.
- **Bag (after grasp)**: rigidly attached to the gripper, `T_obj(t)=T_ee(t)·T_ee←obj`.

→ Perturb the trajectory ⇒ re-derive all poses ⇒ re-render both views. That is
the world model.

## Layered compositional renderer (per view, per timestep)

```
background (static env)  ◁  robot (URDF@joints)  ◁  object (bag, grasp-attached)
```
No 3DGS: both backgrounds come from the **real frames** (point cloud / inpainted
plate), which is why they keep full detail.

- **ZED background** = the real **inpainted clean plate** (fixed camera, robot
  painted out with a SAM mask). The URDF arm is composited over it at the IK joints
  from the calibrated ZED viewpoint (target-free PnP into the base frame).
- **Wrist background** = the real wrist frame lifted to a **depth point cloud**
  (Video-Depth-Anything over the sequence by default — temporally consistent;
  per-frame DA-v2 via `depth.estimator: mono`) and reprojected under the
  per-frame optical-frame shift; the disocclusion holes are **inpainted** (or
  generated by GenWarp).
- **Robot** = exact-camera MuJoCo render (`RobotRenderer`) — third-person view only;
  in the wrist view the gripper is rigid to the camera, so its pixels are copied
  straight from the original frame inside a fixed trapezoid.
- **Bag** = coarse proxy / segmented layer in ZED; higher-fidelity in the wrist view
  where it matters (currently carried by the reprojected real frame).

## Current strategies & options (2026-07-01)

| Choice | Options (`configs/worldmodel.yaml`) | Notes |
|---|---|---|
| Wrist renderer | `wrist_renderer: depthwarp \| genwarp` | point-cloud reproject vs. diffusion NVS |
| Wrist hole-fill | `warp.fill_method: none \| nearest \| bilinear \| edge_aware \| inpaint` | depth-warp path; compare via `scripts/09` |
| Forward-splat | `warp.kernel_splat: true` | 2×2 splat → fewer pin-holes |
| Perturbation envelope | `perturb.envelope: converge_at_grasp \| constant` | reach-same-grasp vs. rigid failure sim |
| Feasibility gate | `perturb.ik_tol` | success/failure label (a2l-pr signal) |
| GenWarp shift | `genwarp.depth_scale / mode / num_inference_steps / guidance_scale` | only when `wrist_renderer: genwarp` |
| Views | `render_wrist`, `render_zed` | toggle either view |

Full knob-by-knob reference: **[synthetic_data.md](synthetic_data.md)**.

## Demonstrated (2026-06-30)

- **Base-perturbation failure simulation, ZED view**
  (`renders/worldmodel/r2s_perturb_zed.png`, `r2s_perturb_exec.mp4`): the same
  recorded reach-grasp rendered with the base shifted left/right/low — the robot
  moves coherently against the fixed scene, so the gripper↔bag misalignment
  ("too far left to grasp") is directly visible. This is the core augmentation +
  failure-simulation loop.
- Robot placement from the calibrated ZED view is accurate enough to be useful
  (small residual from the assumed ZED focal length + green→link6 offset).

## Delivered: end-to-end synthetic-data pipeline (2026-07-01)

The above is now wired into a single package + script that emits synthetic
episodes in the **input dataset format**. See **[synthetic_data.md](synthetic_data.md)**
for the full user guide (all knobs and where to edit them).

- Package `o2m.worldmodel`; driver `scripts/07_synthesize_episode.py`; config
  `configs/worldmodel.yaml`.
- **Converging perturbation**: base-frame EE offset, full during approach →0 at the
  grasp frame →0 after (reaches the same grasp, ends the same way). Auto-detected
  grasp frame.
- **Feasibility label** via IK anchored to FK(measured)+offset → `success` /
  `failure` (a2l-pr signal), stored in `perturbation.json`.
- **Wrist** view = depth-warp (gripper kept fixed; trapezoid halved). **Third-person**
  view = URDF robot @ IK joints over the ZED clean plate.
- **Output** = `actions.csv` (CRLF, only `slave_j*`/`slave_ee_*` overwritten) +
  `realsense_color/` + `zed_color/` + `perturbation.json`.

## Metric scene cloud & novel third-person views (stages 10-13)

- **Stage 10** (`scripts/10_zed_metric_scene.py`, `o2m.worldmodel.scene_cloud`):
  metric depth for the ZED clean plate. Video-Depth-Anything disparity is
  metrified with an affine fitted on **table-plane anchors** (the base sits on
  the table, so every cloth pixel's metric depth follows from the calibrated
  ray-plane intersection) and validated on the FK-tracked green gripper
  (~0.12 m RMSE). Outputs `align/zed_scene_metric.npz` +
  `pointcloud/zed_scene_metric.ply` (base frame).
- **Depth-ordered composite**: with `worldmodel.thirdperson.scene_depth_npz`
  set, the third-person robot composite hides the arm behind nearer scene
  geometry (`depth_margin` absorbs the metric error).
- **Stage 13** (`scripts/13_novel_thirdperson.py`): renders the metric cloud
  from **new camera positions/angles** (`o2m.render.points`) and
  depth-composites the URDF arm — the third-person camera can be moved
  ±25–30° azimuth / ~12° elevation before unseen-surface holes dominate.
  Deliverables in `renders/novel_view/`.
- **Stage 11** (`scripts/11_fuse_wrist_cloud.py`, `o2m.worldmodel.fusion`):
  wrist-video fusion into the same cloud — implemented (FK-posed unprojection,
  triangulated metric scale, trimmed ICP) but currently **blocked**: FK
  `hand_cam` relative poses disagree with image-estimated ones by 10–27°,
  pointing at a camera-mount/joint-convention error. Needs a one-time hand-eye
  calibration; see notes.md (2026-07-02).

## Still open

1. **Refine ZED intrinsics** (solve fx in PnP) → tighter robot alignment.
2. **Hand-eye calibrate the wrist mount** (AX=XB) → unblocks multi-frame wrist
   fusion (stage 11), which then fills disocclusion holes at larger offsets.
3. **Grasp-attached bag** (`T_ee←obj` fit at grasp) so the object follows the
   gripper in both views under perturbation.
